# World of Fantasy

探索型育成RPGテンプレート。プレイヤーが町の施設を発展させながら、異なるエリアを探索して資源を集め、育成するゲームです。

## ゲーム概要

- **ジャンル**: 探索型育成RPG
- **エンジン**: Pyxel（レトロゲームエンジン）
- **開発言語**: Python
- **画面解像度**: 160×120

### ゲームの流れ

1. **町画面（TOWN）**: 施設のアップグレード、ジョブチェンジ、スキル振り分け
2. **エリア生成（CREATE）**: プレイヤーのステータスと町の施設ボーナスでエリアを生成
3. **探索画面（EXPLORE）**: 生成されたエリアを探索し、戦闘・お宝・アクシデントなどのイベントが発生

## プロジェクト構造

```
world_of_fantasy/
├── main.py                 # ゲームのエントリーポイント・画面管理
├── core/
│   ├── engine.py          # ゲーム世界ロジック・エリア・探索システム
│   ├── models.py          # データベース・装備・レシピ・ジョブ定義
│   ├── player.py          # プレイヤークラス・ステータス・スキルツリー
│   ├── town.py            # 町クラス・施設管理・アップグレード
│   └── constants.py       # グローバル定数（予約）
├── assets/                # グラフィック・サウンドなどのリソース
└── README.md              # このファイル
```

## ファイル説明

### `main.py` - アプリケーション本体

| 役割 | 内容 |
|-----|------|
| **画面管理** | 3つの画面状態を管理（TOWN/CREATE/EXPLORE） |
| **入力処理** | キーボード入力をキャプチャしてゲームロジックを実行 |
| **ゲームループ** | `update()` と `draw()` で毎フレーム更新・描画 |

**主なメソッド**:
- `update_town()` - 施設アップグレード、ジョブチェンジ、スキル振り分け処理
- `update_create()` - エリア生成ロジック
- `update_explore()` - 探索ロジック・報酬処理

**操作キー**:
| キー | 機能 |
|-----|------|
| `1` | 町画面へ |
| `2` | エリア生成画面へ |
| `3` | 探索画面へ（エリアが存在する場合） |
| `S` | 施設「学校」をアップグレード |
| `M` | 施設「市場」をアップグレード |
| `J` | ジョブをサイクル切り替え |
| `Z/X/C` | スキル取得（スキルポイントがある場合） |
| `A` | エリア生成 |
| `E` | 探索開始 |

---

### `core/engine.py` - ゲーム世界のロジック

| クラス | 役割 |
|------|------|
| **Area** | エリア（探索対象）を管理するクラス |

**Areaの属性**:
- `struct` - 構造難度（数学）
- `life` - 生命値（生物）
- `control` - 制御難度（物理）
- `difficulty` - 総合難度（これが戦闘の判定基準）
- `reward_base` - 基本報酬（ここから戦闘結果で倍率変動）
- `durability` - 耐久度（探索ごとに1減少）

**探索イベント（確率）**:
| イベント | 確率 | 報酬 |
|---------|------|------|
| 🗡️ **戦闘** | 40% | 金1.5倍 + 経験値20 + 素材1～3 |
| 💎 **お宝** | 30% | 金2倍 + 経験値10 + 素材3～8 |
| 💥 **アクシデント** | 30% | 金0.2倍 + 経験値2 + 素材0～1 |

---

### `core/models.py` - ゲームデータベース

| クラス | 役割 |
|------|------|
| **Equipment** | 装備品（武器・防具）を定義 |
| **Recipe** | クラフトレシピを定義 |
| **ItemDatabase** | 全装備とレシピを一元管理 |
| **Job** | ジョブ（職業）とそのボーナスを定義 |
| **JobDatabase** | 全ジョブを一元管理 |

**ジョブ種類**:
| ジョブ | 説明 | HP | ATK | LOGIC | 初期選択 |
|--------|------|-----|-----|--------|---------|
| WARRIOR | 戦士 | +5 | +3 | 0 | ✓ |
| MAGE | 魔術師 | +1 | 0 | +5 | ✓ |
| THIEF | 盗賊 | +2 | +1 | +2 | ✓ |
| PALADIN | 聖騎士 | +10 | +5 | +3 | ✗ |

**装備例**:
- 木の杖：攻撃力+2
- 鉄の剣：攻撃力+5
- 布の服：攻撃力0
- 革の鎧：攻撃力+2

---

### `core/player.py` - プレイヤーステータス

| 属性 | 説明 |
|-----|------|
| `level` | 総合レベル |
| `gold` | ゲーム内通貨 |
| `material` | クラフト素材 |
| `current_job` | 現在のジョブ |
| `unlocked_jobs` | 選択可能なジョブ list |
| `skill_points` | 振り分け可能なスキルポイント |
| `weapon`, `armor` | 装備中の武器・防具 |
| `base_logic` | 基礎構造理解度（エリア生成に影響） |
| `base_economy` | 基礎経済理解度（報酬に影響） |

**スキルツリー**:
| スキル | 効果 |
|--------|------|
| **analysis** | エリア生成の難度を上げる（より高い報酬） |
| **survival** | 戦闘勝利確率 +2 per level |
| **crafting** | クラフト関連のボーナス |

**レベルアップ**:
- 探索でEXPを獲得
- `next_exp` に到達でレベルアップ
- テンプレート内は `1.2倍` ずつ増加
- レベルアップごとに:
  - スキルポイント +1
  - base_logic +1

---

### `core/town.py` - 町の施設管理

| 施設 | ボーナス効果 |
|-----|------------|
| **school** | エリア生成時の `struct` にボーナス → 高い報酬 |
| **market** | 探索報酬 × (1 + 0.1 × level) |
| **lab** | 将来実装用（予約） |
| **workshop** | クラフト素材消費を軽減 |

**アップグレード**:
- **ゴールドコスト** = `100 × (レベル)²`
- **素材コスト** = `レベル × 2`

## 拡張のポイント

このテンプレートを拡張する際の実装ポイント：

### 新しいジョブの追加
**ファイル**: `core/models.py` の `JobDatabase.JOBS`

```python

```

### 新しいエリアタイプの追加
**ファイル**: `main.py` と `core/engine.py`

異なる `struct/life/control` の組み合わせで、エリアの特性を変更できます。

### クラフトシステムの実装
**ファイル**: `core/player.py` と `core/models.py`

- `equip(item)` で装備の切り替え
- `get_craft_bonus()` で消費素材を軽減

### セーブ・ロード機能
**ファイル**: 新規 `core/save.py`

プレイヤーとタウンの状態をJSONで保存・復元します。

## 操作ガイド

### 初期画面：町（TOWN）

施設をアップグレードしてプレイヤーを成長させます。

```
[TOWN] GOLD:500 LV:1
学校をアップグレード (S)
市場をアップグレード (M)
ジョブ切り替え (J)：WARRIOR
スキル振り分け (Z/X/C)：ANALYSIS / SURVIVAL / CRAFTING
→ エリア生成画面へ (2)
```

### エリア生成（CREATE）

町の施設ボーナスを受けながら、エリアを生成します。

```
Logic = base_logic + school_bonus + analysis_skill
→ エリアデータ（struct, life, control）が決定
→ 難度・報酬が計算される
→ 探索画面では最大3回まで探索可能（durability）
```

### 探索画面（EXPLORE）

生成されたエリアを探索し、イベントを進行させます。

```
[RUIN] DURABILITY: 3/3
タイマー進行中...
→ 「MONSTER BATTLE」「TREASURE FOUND」などのイベント
→ 報酬獲得
→ 次のエリア生成ボタンで繰り返し
```

### 難度調整

- 各イベント（戦闘0.4→0.3、報酬1.5→2.0など）の確率・倍率を変更

- `Player.next_exp` の初期値と倍率（現在`1.2倍`）を調整
- `Event.exp` の獲得量を変更
